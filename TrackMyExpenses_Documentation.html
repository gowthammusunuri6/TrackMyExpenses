📋 Project Overview

TrackMyExpenses is a comprehensive, feature-rich web application designed to help users track their spending, manage budgets, and gain AI-powered financial insights. The application combines modern web technologies with intelligent analytics to provide users with a powerful tool for personal finance management.

Key Vision: Empower users to take control of their finances through intuitive tracking, real-time analytics, and AI-driven insights.

🔌 API Documentation

Authentication Endpoints

POST /api/signup
Create a new user account
POST /api/login
Authenticate user with email and password
POST /api/logout
End user session
GET /api/user/:id
Retrieve user profile information
PUT /api/user/:id
Update user profile (name, email, password, budget)

Expense Endpoints

POST /api/expenses
Add a new expense (amount, category, date, paymentMode, notes)
GET /api/expenses/:userId
Retrieve all expenses for a specific user
PUT /api/expenses/:expenseId
Update an existing expense
DELETE /api/expenses/:expenseId
Delete an expense record

Analytics Endpoints

GET /api/analytics/:userId
Retrieve comprehensive analytics (totals, trends, suggestions)
GET /api/savings/:userId
Get savings goals and progress for user
POST /api/savings
Create new savings goal

Chatbot Endpoints

POST /api/chatbot/message
Send message to AI chatbot and receive response

Admin Endpoints

GET /api/admin/users
Get all users (admin only)
GET /api/admin/all-expenses
Get all expenses across all users (admin only)
POST /api/admin/export
Export data to Excel format (admin only)

🔄 Data Flow & Processing

Expense Addition Flow

Frontend:
1. User fills expense form (amount, category, date, payment mode, notes)
2. Form validation on client-side
3. Send POST request to /api/expenses

Backend:
4. Receive and validate data
5. Create unique ID (UUID)
6. Add timestamp
7. Write to Excel database
8. Return success response

Frontend:
9. Display success message
10. Refresh expense list
11. Update analytics/charts
12. Clear form fields

Analytics Calculation Process

Data Collection:
• Fetch all user expenses from Excel

Calculations:

Visualizations:

🔒 Security & Best Practices

Security Measures

Security Aspect Implementation Protection
Password Security bcrypt hashing (salt rounds: 10) Protects against rainbow table attacks
Session Management express-session with secure cookies 24-hour expiry, prevents session hijacking
CORS Policy CORS middleware enabled Control cross-origin requests
Input Validation Server-side validation on all endpoints Prevents malicious data injection
File Upload Security filetype & size validation via Multer Only allows image uploads ≤5MB
Error Handling Generic error messages to clients Prevents information disclosure

Best Practices Implementation

  • State Management: Session storage for client-side user data
  • Code Organization: Modular route handlers (auth, expenses, analytics)
  • Error Handling: Try-catch blocks with meaningful error messages
  • API Design: RESTful endpoints with proper HTTP methods
  • Frontend Validation: Client-side form validation before submission
  • Responsive Design: Mobile-first CSS with media queries
  • Performance: Efficient database queries and chart rendering
  • Accessibility: Semantic HTML, ARIA labels, keyboard navigation

⚡ Performance Optimization

Frontend Optimization

  • Chart Rendering: C3.js provides efficient chart updates without full re-renders
  • Session Storage: User data cached in browser to reduce API calls
  • Lazy Loading: Charts only render when page is visible
  • CSS Optimization: Separate stylesheets for each page to reduce load
  • Event Delegation: Single event listeners for dynamic content
  • Async Operations: Non-blocking API calls with async/await

Backend Optimization

  • Excel Caching: In-memory data processing reduces I/O
  • Index Optimization: Efficient array filtering by User ID
  • Response Compression: JSON payloads minimized
  • Error Handling: Prevents unnecessary database queries
  • Connection Reuse: Single Express app instance

Common Issues & Solutions

Issue Cause Solution
Port 3000 in use Another process using the port Change PORT in server.js or kill existing process
Module not found Missing dependencies Run: npm install
Session not persisting Browser storage disabled Enable sessionStorage/localStorage in browser
Charts not rendering C3.js library not loaded Check browser console for errors
File upload failing Upload directory doesn't exist Multer creates directory automatically
Login failing Incorrect credentials or no users Create account or use demo credentials
